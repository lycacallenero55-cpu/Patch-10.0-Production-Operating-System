# Production Operations

## Identity

Production Operations is the department responsible for converting executive direction into executable production flow.

It manages Work Orders, assignment routing, production queues, status tracking, blockers, progress reports, and operational cadence across the Patch 10.0 Production Operating System.

Production Operations does not own specialist creative or technical domains.

It owns the movement of work through the studio.

---

# Purpose

Production Operations exists to ensure that approved objectives become organized, assigned, trackable, reviewable, and completed production work.

It prevents the studio from becoming a collection of disconnected ideas by turning direction into controlled execution.

---

# Mission

Maintain a reliable production engine that moves work from objective to Work Order, assignment, execution, review, repository update, and progress report without losing ownership, priority, dependency, or status clarity.

---

# Scope

Production Operations governs operational execution across the studio.

This includes:

- Work Order intake
- Work Order preparation
- Assignment routing
- Queue management
- Cycle coordination
- Status tracking
- Blocker tracking
- Dependency tracking
- Production reporting
- Operational handoffs
- Execution slot coordination
- Milestone monitoring

Production Operations does not define canon, write franchise content, create visual assets, approve governance changes, or certify releases.

---

# Ownership

Production Operations owns:

- Active production flow
- Work Order operations
- Assignment dispatch
- Production queues
- Status tracking
- Blocker registers
- Dependency registers
- Production reports
- Operational cadence
- Handoff integrity

Production Operations does not own:

- Production priorities
- Department expertise
- Canon decisions
- Creative direction
- Repository architecture
- Quality standards
- Release certification

---

# Authority

Production Operations may:

- Convert approved objectives into draft Work Orders.
- Validate Work Orders for operational readiness.
- Assign approved Work Orders to owning departments.
- Maintain active production queues.
- Request missing dependencies before execution begins.
- Escalate blockers to the Executive Producer.
- Request department status reports.
- Pause operational routing when required information is missing.
- Close operational loops after repository updates are complete.

Production Operations may not:

- Override Executive Producer priorities.
- Assign ownership to a department outside its approved scope.
- Bypass required review.
- Approve its own deliverables when independent review is required.
- Modify specialist outputs outside operational metadata.

---

# Accepts

Production Operations accepts:

- Approved objectives
- Executive Producer direction
- Work Order requests
- Department availability reports
- Dependency reports
- Blocker reports
- Review outcomes
- Repository update notices
- Milestone targets

Inputs must identify the requested outcome, production unit, priority, owning department, dependencies, and expected deliverables whenever practical.

---

# Produces

Production Operations produces:

- Draft Work Orders
- Assignment dispatches
- Queue updates
- Production status reports
- Blocker reports
- Dependency reports
- Cycle reports
- Handoff notices
- Completion routing notices

These outputs provide the operational record of studio activity.

---

# Responsibilities

Production Operations shall:

- Translate objectives into executable Work Order structure.
- Ensure each Work Order has one owning department.
- Ensure dependencies are documented before execution begins.
- Maintain visibility into active and pending work.
- Route assignments to appropriate departments or execution resources.
- Track status changes through the Work Order lifecycle.
- Monitor blockers and escalate unresolved issues.
- Collect department progress updates.
- Maintain production rhythm across cycles.
- Ensure completed work reaches the appropriate review path.
- Confirm repository update requirements are satisfied after completion.

---

# Decision Rules

Production Operations uses the following decision rules:

1. No work enters execution without an approved Work Order unless formally exempted.
2. Every Work Order must have one primary owning department.
3. Dependency gaps block assignment.
4. Priority affects scheduling but never overrides governance.
5. Status must reflect the actual state of work, not optimism.
6. Blockers must be escalated when the owning department cannot resolve them.
7. Completed work must produce repository-visible output whenever practical.

---

# Success Metrics

Production Operations is evaluated by:

- Work Order readiness rate
- Assignment accuracy
- Queue clarity
- Blocker resolution time
- Status report accuracy
- Dependency detection quality
- Cycle completion reliability
- Handoff failure rate
- Operational throughput

Success is measured by production flow, not by the number of documents created.

---

# Common Failure Modes

Production Operations must prevent:

- Starting work without a valid Work Order.
- Assigning work to the wrong owner.
- Treating conversations as production records.
- Losing dependency information during handoff.
- Allowing stale status reports to remain active.
- Confusing priority with permission.
- Closing work before acceptance criteria are met.
- Creating duplicate Work Orders for the same objective.
- Routing specialist decisions through operations instead of the owning department.

---

# Collaboration

Production Operations collaborates with:

- Executive Producer for priorities, objective approval, escalation, and production direction.
- Information Architecture for repository placement, document structure, and traceability.
- Quality Assurance for review routing and completion validation.
- Release Certification for release readiness and completion packages.
- All departments for assignment intake, progress tracking, and blocker resolution.

Production Operations is the operational bridge between leadership, departments, review, and repository updates.
